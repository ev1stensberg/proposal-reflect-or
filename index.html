Reflect.logical[AND|OR]

1Reflect.logicalOR ( object, object, [value] )#

When the logicalOR method is called with arguments, the following steps are taken:

  1. Let Reflect be an **object**.
    1. Let object be an **object**.
    2. Let object be an **object**.
    3. Let [value] be an **primitive** that returns null or undefined based on what the user inputs.
    4. Throws an TypeError if object is non-existent.
  2. Assigns [value] as undefined if parameter isn't supplied.
    1. Calculations happens through normal OR logic rules.
    2. Return a the new value upon the "Reflected" object, of which, the user assigns.

2Reflect.logicalOR ( object, object, [value]#

When the logicalAND method is called with arguments, the following steps are taken:

  1. Let Reflect be an **object**.
  2. Let object be an **object**.
  3. Let object be an **object**.
  4. Let [value] be an **primitive** that returns null or undefined based on what the user inputs.
  5. Throws an TypeError if object is non-existent.
  6. Assigns [value] as undefined if parameter isn't supplied.
  7. Calculations happens through normal AND logic rules.
  8. Return a the new value upon the "Reflected" object, of which, the user assigns.
